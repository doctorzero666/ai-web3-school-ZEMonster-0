## 今天的三份深度研究

### 1. Claude Managed Agent 记忆系统

> 笔记：[[Claude-Managed-Agent-记忆系统深度拆解]]

**对 AgentPay 的启发**：
- **文件优先，不用 Embedding** → AgentPay 的用户消费记录用结构化 JSON/Markdown 存储，不用向量数据库
- **Dreaming 机制** → AgentPay 可以后台扫描 Intent 执行记录，自动生成消费报告
- **FUSE 桥接** → AgentPay 前端看到的是「消费记录」，底层是链上 Job 数据 + IPFS 哈希

### 2. Claude Code SubAgent 原理

> 笔记：[[ClaudeCode-SubAgent原理及面试真题]]

**对 AgentPay 的启发**：
- **FORK + Prompt Cache** → AgentPay 多个 SubAgent（搜索/验证/报告）共享前缀，省推理成本
- **不同任务不同模型** → 搜索 API 用便宜模型、风险评估用强模型
- **.md 文件定义 Agent** → 用户的 Payment Intent 可以持久化为 .md 文件，跨 session 复用

### 3. Top 10 顶级交易员策略

> 笔记：[[Top10顶级交易员策略总结]]

**对 AgentPay 的启发**：
- **仓位管理 = 预算控制** → 同样的保命逻辑
- **胜率不重要，盈亏比重要** → AgentPay 不追求每笔交易完美，追求「用户总资产安全」
